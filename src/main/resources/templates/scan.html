하드코딩 탐지 — 사용 가이드

이 도구를 왜 만들었나요?

소스코드에 직접 박힌 값(하드코딩)과 고객사 전용 분기 로직은 유지보수성을 낮추고 배포 오류를 유발합니다.

  • 고객사 코드가 코드 곳곳에 박혀 있어 다른 고객사 적용 시 누락 발생
  • 상태값·역할명 문자열 비교가 DB 코드와 불일치할 때 런타임 오류
  • 설정으로 빼야 할 값이 코드 안에 있어 배포 없이는 변경 불가
이 도구는 자동 수정기가 아닙니다.
정적 분석으로 "이 줄이 문제일 수 있으니 확인해 주세요" 라고 알려줍니다.
최종 판단과 리팩토링은 개발자가 직접 합니다.

사용 방법

  • 소스코드를 코드 입력창에 붙여넣거나 파일을 업로드합니다
  • 언어를 선택합니다 (Java 기본, JS/TS, SQL, Config, 텍스트)
  • 파일명을 입력하면 공통 영역 가중치(WT-001)가 적용됩니다
  • 탐지 키워드에 프로젝트 고객사 코드를 등록하면 추가 규칙이 적용됩니다
  • 제외 키워드에 의도적 하드코딩 값을 등록하면 결과에서 제외됩니다
  • 스캔 실행 버튼을 클릭합니다

지원 파일 형식: .java · .js · .ts · .sql · .yml · .properties · .json · .txt · .md · .docx · .pptx · .pdf(텍스트형)

규칙 카테고리

카테고리 규칙 탐지 내용
HCHC-001 ~ HC-008코드에 직접 박힌 값 — 템플릿 ID, 상태코드, 역할명, 고객사 코드, 매직 넘버, URL, SQL 고정값, 승인 단계값
CLCL-001 ~ CL-008고객사 특화 로직 — 고객사 코드 기준 조건문, 조직별 예외 분기, 전용 클래스명, 전용 주석, 반복 분기 패턴
WTWT-001 ~ WT-005점수 가중치 — 공통 영역 파일 +2, 테스트 코드 제외, static final -2, 중첩 깊이 +1, 반복 규칙 +1

결과 보는 법 — 심각도

심각도 점수 의미
HIGH8점 이상즉시 검토 권장 — 고객사 분기, URL 하드코딩 등 설정화 필요
MEDIUM5 ~ 7점검토 권장 — 상태코드·역할명 직접 비교, SQL 고정값
LOW2 ~ 4점가볍게 확인 — 전용 주석, 반복 분기 패턴
INFO0 ~ 1점참고 — 낮은 위험, 맥락에 따라 무시 가능

탐지 키워드 vs 제외 키워드

  • 탐지 키워드 — 프로젝트에서 사용하는 고객사 코드(예: C001, CORP_A)를 등록하면 HC-004, CL-001 규칙에서 추가로 탐지합니다
  • 제외 키워드 — 의도적으로 하드코딩된 값(예: LOCAL, TEST_MODE)을 등록하면 해당 값이 matchedText에 포함된 결과를 결과에서 제외합니다

※ 제외 키워드는 완전 일치가 아니라 포함 여부로 판단합니다.
