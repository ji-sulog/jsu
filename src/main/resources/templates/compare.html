요구사항 변경 검토 보조 — 사용 가이드

이 도구를 왜 만들었나요?

요구사항 문서는 프로젝트가 진행되는 동안 계속 바뀝니다. 그런데 바뀐 문서를 받았을 때 이런 문제가 반복됩니다.

  • 어디가 바뀌었는지 빠르게 파악하기 어렵다
  • 단순 문구 수정인지, 실제 기능·정책 변경인지 구분하기 어렵다
  • 권한, 숫자, 예외 조건 같은 중요한 변경을 놓칠 수 있다
  • 놓친 변경이 재작업·장애·커뮤니케이션 비용으로 이어진다
이 도구는 자동 판정기가 아닙니다.
맞춤법 검사기처럼 — "이 부분이 중요할 수 있으니 먼저 봐주세요" 라고 추천만 합니다.
최종 판단은 항상 사람이 합니다.

사용 방법

  • 이전 요구사항 칸에 기존 문서 텍스트를 붙여넣거나 파일을 업로드합니다
  • 최신 요구사항 칸에 새 문서 텍스트를 붙여넣거나 파일을 업로드합니다
  • 변경 사항 비교하기 버튼을 클릭합니다
  • 결과 목록을 위에서부터 순서대로 검토합니다

지원 파일 형식: .txt · .md · .docx · .pptx · .pdf(텍스트형). 스캔 PDF는 지원하지 않습니다.

결과 보는 법 — 중요도

각 변경 항목은 규칙 엔진이 점수를 계산해 중요도를 추천합니다.

중요도점수의미주요 감지 패턴
🔴 높음7점 이상먼저 확인 권장권한 변경, 의미 반전(가능↔불가), 상태·절차 변경
🟡 확인 필요4 ~ 6점검토 권장숫자·기간 변경, 필수 여부 변경, 예외·조건 변경
🔵 낮음1 ~ 3점가볍게 확인중요 섹션 내 소규모 변경, 중요 문장 삭제
⚪ 일반0점단순 문구 수정 가능성동의어 치환, 어미·조사 수준 변경

※ 중요도는 추천값입니다. 실제 영향 여부는 담당자가 직접 판단해야 합니다.

결과 보는 법 — 이유 태그

  • 주체 또는 권한 관련 변경 — 사용자/관리자/권한 등 주체가 바뀜
  • 허용 여부 변경 — 가능/불가/허용/금지 등 변경
  • 필수 여부 변경 — 필수/선택 여부 변경
  • 숫자 또는 제한값 변경 — 수치·횟수·기간 변경
  • 상태 또는 절차 변경 — 승인·반려·단계 변경
  • 예외 또는 조건 변경 — 예외·조건·단서 조항 변경
  • 중요 문장 삭제 (삭제 가중치) — 중요 키워드가 포함된 문장이 삭제됨. 권한·예외 삭제 +3, 상태·필수 삭제 +2
  • 의미 반전: A ↔ B — 가능↔불가, 포함↔제외, 필수↔선택 등 정책 방향 반전 감지
  • 권한·조건 반전 복합 감지 — 의미 반전 + 권한/예외 조건 동시 발생 시 추가 +2점
  • 필수 여부 반전 복합 감지 — 의미 반전 + 필수/선택 키워드 동시 발생 시 추가 +1점

결과 보는 법 — 검토 체크 항목

각 카드 하단의 버튼을 클릭해 검토 상태를 표시할 수 있습니다.

  • 중요 — 이 변경이 중요하다고 판단됨
  • 개발 반영 필요 — 코드·로직 수정이 필요함
  • 운영 반영 필요 — 운영 정책·설정 변경이 필요함
  • 단순 문구 수정 — 의미 변화 없는 표현 수정으로 판단
  • 확인 완료 — 검토를 마쳤음

※ 체크 상태는 현재 세션에서만 유지됩니다.
