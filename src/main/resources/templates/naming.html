네이밍 컨벤션 검사 — 사용 가이드

이 도구를 왜 만들었나요?

네이밍은 코드 리뷰에서 가장 자주 지적되면서도, 실수가 반복되는 영역입니다.

이 도구는 단순히 "문법상 맞나?"만 보는 lint가 아닙니다.
"이 이름이 읽기 좋은 이름인가?"까지 같이 봅니다.

사용 방법

  • Java 코드를 코드 입력창에 붙여넣거나 .java / .txt 파일을 업로드합니다
  • 필요에 따라 의미 품질 규칙과 boolean 이름 규칙을 토글합니다
  • 검사 실행 버튼을 클릭합니다
  • 결과를 심각도 / 카테고리 필터로 좁혀서 확인합니다

의미 품질 규칙 (NM-005 ~ NM-008)

규칙 내용 예시 문제
NM-005boolean 접두사 (is/has/can/should 등)boolean login → canLogin
NM-006의미가 약한 이름 경고 (data, tmp, obj 등)String data → userJson
NM-007컬렉션 이름 품질 (복수형/접미사 권장)List<User> data → users
NM-008축약어 과다 (토큰의 60% 이상이 3자 이하)usrDtlCdVal
의미 품질 규칙(NM-005~008)은 강제가 아닌 권장입니다.
일부 이름은 맥락에 따라 적절할 수 있으므로 팀의 판단으로 수용 여부를 결정하세요.
