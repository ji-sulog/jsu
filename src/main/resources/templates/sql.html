SQL 품질 검사 — 규칙 가이드
SQ01 SELECT * — 불필요한 컬럼 조회로 네트워크·메모리 비용 증가. 필요한 컬럼만 명시.
SQ02 WHERE 없는 UPDATE — 전체 행 변경 위험. 의도적 전체 갱신이 아니면 WHERE 필수.
SQ03 WHERE 없는 DELETE — 전체 행 삭제 위험. WHERE 조건 반드시 추가.
SQ04 앞 와일드카드 LIKE — LIKE '%값' 형태는 인덱스 풀스캔 유발. LIKE '값%' 또는 전문 검색 고려.
SQ05 페이지네이션 누락 — LIMIT / ROWNUM / FETCH NEXT 없는 대량 조회. 메모리 초과·지연 위험.
SQ06 서브쿼리 중첩 — 인라인 서브쿼리는 N+1 문제 유발 가능. JOIN 또는 WITH(CTE) 리팩터링 고려.
SQ07 OR 조건 과다 — 3개 이상 OR는 인덱스 활용 어려움. IN 절이나 UNION 분리 고려.
SQ08 커스텀 금지 패턴 — 팀에서 직접 등록한 금지 키워드 감지.
